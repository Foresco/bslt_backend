# Порядок тестирования запросов
Просмотр возможностей pytest
```commandline
pytest --help
```

Запуск теста из конкрентного файла
По умолчанию pytest ищет тесты в каталоге запуска и всех подкаталогах. 
Он ищет файлы, начинающиеся с test_ или заканчивающиеся на _test.
Чтобы выполнить только наши новые тесты задач, вы можете предоставить pytest все имена файлов, которые вы хотите запустить, или каталог, или вызвать pytest из каталога, где находятся наши тесты.

* Тестовые файлы должны быть названы test_<something>.py или <something>_tests.py. (указано в pytest.inu)
* Методы и функции тестирования должны быть названы test_<something>.
* Тестовые классы должны быть названы Test<Something>.

Запуск только одного теста.
Необходимо указать имя теста после ::
```commandline
pytest tasks/test_four.py::test_asdict
```

## Пропуск тестов
Вы можете заставить pytest пропустить тест, используя декораторы @pytest.mark.skip() или pytest.mark.skipif()
