# Инструментарий наследования данных из Базальты версии 1

У каждой модели, предназначенной для экспорта, должен быть метод **to_dict**, 
формирующий словарь со свойствами для импорта каждой строки.
Дополнительно должен быть класс Exporter, возвращающий модели, предназначенные для экспорта.

Исходный массив подключается в settings в разделе DATABASES
Переключение на него обспечивает прописанная там же DATABASE_ROUTERS

Перечень выгружаемых моделей привеодится в методе list_to_export класса Exporter в исходном модуле models.

Запускается командой
```commandline
python manage.py import_legacy 1 
```

1 - идентификатор сессии для привязки импортируемых данных (может быть другим)

Дополнительные параметры:

--source Имя базы данных-источника

--package Номер пакета, который нужно импортировать

--from Номер пакета, начиная с которого нужно импортировать

--check_only Не выполнять импорт, только проверить выгрузку. Будет сформирован json-файл в корневом каталоге.
